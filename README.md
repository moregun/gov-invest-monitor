# 🏛️ gov-invest-monitor

**国家队宽基ETF资金自动监控系统**

通过追踪核心宽基ETF的份额变动、溢价率与资金流向，基于多维度特征模型自动识别国家队（汇金/证金体系）护盘资金动向，支持 GitHub Actions 定时运行与可视化面板展示。

| 功能 | 说明 |

| 📊 **多标的监控** | 覆盖上证50、沪深300、中证500、科创50、中证1000、创业板ETF 6只核心宽基 |
| 🔍 **智能识别** | 4维度模型识别国家队特征：同步流入、持续申购、溢价异常、逆周期操作 |
| 📈 **资金测算** | 基于份额变动 × 净值估算净流入，对标高盛等机构统计口径 |

| 📝 **日报生成** | 自动输出 Markdown 格式每日分析报告 |

## 🧠 监控原理

国家队（汇金/证金）操作具有显著的行为特征，本项目基于以下维度交叉判定：

### 四大识别维度

1. **分散布局特征**
   - 同时大额买入多只核心宽基ETF
   - 不集中单一行业/赛道，目标是托住全市场指数
   - 阈值：≥4只宽基同步流入

2. **持续性特征**
   - 连续多日逆市申购（通常3天以上）
   - 而非游资单日脉冲式进出
   - 阈值：连续3日以上份额正增长

3. **一级市场申购特征**
   - ETF持续正溢价（市价 > 净值IOPV）
   - 反映大额一级市场申购需求
   - 阈值：溢价率 > 0.05% 且多只同步

4. **逆周期操作特征**
   - 市场大跌阶段大额流入
   - 上涨行情暂停买入
   - 结合大盘行情辅助判定

### 信号评级

| 评分区间 | 等级 | 含义 |
|----------|------|------|
| 80-100 | 🔴 强护盘信号 | 多维度同时触发，国家队大规模进场概率高 |
| 50-79 | 🟠 中度护盘信号 | 多个特征符合，存在明显护盘动作 |
| 25-49 | 🟢 弱护盘信号 | 部分特征显现，小规模资金流入 |
| 0-24 | ⚪ 无明显信号 | 暂无国家队操作特征 |

## 🚀 快速开始

### 本地运行

```bash
# 1. 克隆项目
git clone https://github.com/moregun/gov-invest-monitor.git
cd gov-invest-monitor

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行全部任务（抓取 + 分析 + 生成面板）
python src/monitor.py --all

# 4. 查看输出
# 可视化面板: output/dashboard.html
# 分析报告: output/daily_report.md
# 历史数据: data/etf_daily_data.csv
```

### 命令行参数

```bash
python src/monitor.py [选项]

选项:
  --realtime     仅抓取实时行情快照
  --daily        仅更新每日份额汇总数据
  --report       仅生成分析报告
  --dashboard    仅生成可视化面板
  --all          执行全部任务（推荐）
```

## ⚙️ GitHub Actions 部署

### 步骤

1. **Fork 本仓库**到你的 GitHub 账号

2. **启用 GitHub Pages**（可选，用于在线查看面板）
   - 进入仓库 Settings → Pages
   - Source 选择 "GitHub Actions"

3. **工作流自动运行**
   - 已配置 `/.github/workflows/monitor.yml`
   - 每个交易日收盘后（北京时间 15:10 = UTC 07:10）自动执行一次完整抓取 + 分析 + 部署
   - 如需盘中多次快照，可在工作流 `schedule` 中自行追加 cron（见下）

4. **手动触发**
   - Actions → ETF Monitor → Run workflow
   - 可选择运行模式：all / realtime / daily

### 定时配置说明

```yaml
# Cron 表达式（UTC时间，北京时间 = UTC+8）
- cron: '10 7 * * 1-5'  # 周一至周五 15:10 收盘汇总 + 部署面板（默认）
# 如需盘中多次快照，在 schedule 下追加：
- cron: '0 2 * * 1-5'   # 周一至周五 10:00
- cron: '30 3 * * 1-5'  # 周一至周五 11:30
- cron: '0 6 * * 1-5'   # 周一至周五 14:00
```

> 💡 节假日自动无数据，不会产生错误，脚本会优雅处理空数据情况。

编辑 `config.py` 可调整监控参数：

ETF_MONITOR_LIST = [
    {"code": "510300", "name": "沪深300ETF", "exchange": "sh", "category": "宽基核心"},

# 调整判定阈值

    "premium_rate": 0.05,       # 溢价率阈值(%)
    "share_growth_days": 3,     # 连续增长天数

}

## 📊 数据源说明

全部为**公开免费接口，无需 API Key**，已实测验证可用（2026-07 复核）。

| 数据类型 | 数据源 | 状态 | 更新频率 |
|----------|--------|------|----------|
| 实时行情（价格、涨跌） | 新浪财经 hq.sinajs.cn（主）+ 腾讯财经 qt.gtimg.cn（备用自动降级） | ✅ 稳定可用 | 实时 |
| 净值 / IOPV 近似（溢价率计算） | 天天基金盘中估值 fundgz.1234567.com.cn | ✅ 稳定可用 | 实时 |
| 每日资金净流入（资金测算 / 国家队识别核心） | 东方财富 push2his 资金流日K `fflow/daykline` | ✅ 稳定可用 | T+1（收盘后） |
| 资金流入估算 | 直接取「主力净流入」(元) ÷ 1e8 → 亿元 | 依赖上述资金流接口 | 每日计算 |

> 四大识别维度（同步流入 / 持续净流入 / 溢价异常 / 逆周期）现已**全部可用**，
> 不再依赖任何需鉴权的接口。历史份额接口（东方财富 `fundMnfh`）已于 2024 年起失效，
> 本项目改以「每日主力资金净流入」作为国家队识别的核心输入——它比「份额×净值估算」
> 更直接、更准确，且同样免费、无需鉴权。

### 数据口径说明

- **资金净流入**：取自东方财富资金流日K 的 `f52`（主力净流入，单位元），正为净流入、负为净流出。
  该指标直接反映 ETF 申购/赎回方向的资金动向，是国家队（大额净申购）识别的最佳代理变量。
- **份额变动估算**：`资金净流入(元) ÷ 净值(元/份)`，仅作参考，用于匡算份额增减幅度。
- **连续净流入天数**：从最近交易日向前数，遇「非净流入日」即中断（尾随统计）。
- **周期窗口**：采用「最近 N 个交易日」口径（按行数取），而非日历日相减，避免节假日错位。

> 所有数据均来自公开接口，无需额外 Key；若使用需鉴权接口请自行配置凭证。
